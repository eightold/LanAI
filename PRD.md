# Xiaomi Watch S3 AI Chat App PRD

> 基于 Xiaomi Vela JS + DeepSeek / GLM API 的轻量级 AI 聊天应用产品需求文档

---

# 一、产品概述

## 1.1 产品名称（暂定）

**LanAI**

---

## 1.2 产品背景

随着 AI 助手逐渐普及，用户希望在更轻量、更便捷的设备上快速使用 AI 能力。智能手表具备“随时查看、快速交互”的特点，非常适合作为 AI 聊天入口。

本产品计划基于 Xiaomi Watch S3 与 Xiaomi Vela JS 开发一个轻量级 AI 聊天应用，通过接入 DeepSeek / GLM API，实现：

* AI 问答
* 历史会话管理
* 新建聊天
* 删除聊天
* 手表端文字输入

重点强调：

* 简洁
* 快速
* 可用
* 低功耗

不追求复杂 Agent、多模态、插件系统等能力。

---

## 1.3 产品目标

### 核心目标

打造一个：

* 能在手表上快速提问 AI
* 能查看历史聊天
* 操作简单
* 响应速度可接受
* UI 适配圆形表盘

的轻量 AI Chat App。

---

## 1.4 目标用户

### 核心用户

* 小米 Watch S3 用户
* 对 AI 有兴趣的技术爱好者
* 希望随时快速提问 AI 的用户
* 开发者 / 学生 / 极客用户

---

### 用户场景

| 场景     | 描述            |
| ------ | ------------- |
| 快速问答   | 不拿手机，直接手表问 AI |
| 临时记录   | 让 AI 帮忙总结、翻译  |
| 碎片时间使用 | 通勤、走路时快速查看回复  |
| 技术测试   | AI + 智能手表体验探索 |

# 2.2 功能详细说明

---

## 2.2.1 会话列表页（Chat List）

### 功能描述

展示用户所有历史聊天会话。

---

### 功能点

#### 1）展示会话列表

显示：

* 会话标题（自动生成）
* 最后一条消息摘要
* 更新时间

---

#### 2）点击进入聊天

点击会话进入 Chat 页面。

---

#### 3）新建聊天

点击 “New Chat”：

* 创建新会话
* 跳转聊天页

---

#### 4）删除聊天

支持：

* 长按删除
* 或右滑删除

删除后：

* 本地数据同步清除

---

### 数据结构建议

```js
{
  id: "chat_001",
  title: "如何学习 JavaScript",
  updateTime: 1710000000,
  messages: []
}
```

---

## 2.2.2 聊天页面（Chat Page）

### 功能描述

用户与 AI 对话的核心页面（支持 DeepSeek / GLM）。

---

### 功能点

#### 1）消息展示

支持：

* 用户消息
* AI 消息

区分样式：

| 类型 | 对齐 |
| -- | -- |
| 用户 | 右侧 |
| AI | 左侧 |

---

#### 2）消息滚动

支持：

* 自动滚动到底部
* 手势滑动查看历史消息

---

#### 3）文字输入

支持：

* 手表软键盘
* 拼音输入
* 英文输入

输入后点击发送。

---

#### 4）发送消息

流程：

```text
输入文字
 → 点击发送
 → 请求 AI API（按当前 Provider 路由）
 → 返回结果
 → 渲染 AI 回复
```

---

#### 5）AI 回复状态

请求中显示：

```text
Thinking...
```

或 loading 动画。

---

#### 6）错误处理

包括：

* 网络错误
* API 超时
* API Key 无效

提示：

```text
Network Error
Please retry
```

---

## 2.2.4 AI API 接入

### 功能描述

通过 HTTPS 请求 AI Chat API。支持双 Provider：

| Provider | Endpoint | 模型 |
| -------- | -------- | ---- |
| DeepSeek | `https://api.deepseek.com/chat/completions` | `deepseek-v4-flash` |
| GLM（智谱） | `https://open.bigmodel.cn/api/paas/v4/chat/completions` | `glm-5.1` |

---

### 请求流程

```text
Watch App
   ↓ 读取 MODEL_PROVIDER + 对应 API Key
DeepSeek API / GLM API
   ↓
返回 AI 回复
```

---

### API Endpoint

```text
POST /chat/completions
```

---

### Request 示例

DeepSeek（`max_tokens: 300`）：

```json
{
  "model": "deepseek-v4-flash",
  "messages": [
    {
      "role": "user",
      "content": "你好"
    }
  ],
  "max_tokens": 300
}
```

GLM（`max_tokens: 1024`，因 reasoning 模型需更大配额）：

```json
{
  "model": "glm-5.1",
  "messages": [
    {
      "role": "user",
      "content": "你好"
    }
  ],
  "max_tokens": 1024
}
```

---

### Header

```text
Authorization: Bearer <对应Provider的API_KEY>
Content-Type: application/json
```

# 五、技术栈建议

---

## 5.1 前端

### 推荐

* Xiaomi Vela JS

用于：

* 页面开发
* UI 渲染
* 设备交互

---

## 5.2 UI 框架建议

建议：

* 原生组件优先
* 避免复杂第三方库

原因：

* 手表性能有限
* 降低内存占用

---

## 5.3 网络请求

使用：

```js
fetch()
```

或：

```js
request()
```

发送 HTTPS 请求。

---

## 5.4 数据存储

推荐：

```text
Local Storage（JSON）
```

原因：

* 简单
* 足够满足 MVP

---

## 5.5 API 层

推荐封装：

```text
/api
  ├── deepseek.js    (多 Provider 路由，reasoning 回退解析)
  ├── storage.js
  └── chat.js
```

---

## 5.6 项目结构建议

```text
src/
 ├── pages/

 │    ├── chat/           (聊天页)

 │    ├── settings-model/ (Provider 切换)

 │

 │    ├── message-item/

 │

 └── app.js
```

# 六、非功能性需求

---

## 6.1 性能要求

### 启动速度

| 指标   | 要求      |
| ---- | ------- |
| 冷启动  | ≤ 2s    |
| 页面切换 | ≤ 300ms |

---

### 响应时间

| 项目     | 要求    |
| ------ | ----- |
| 输入响应   | 即时    |
| API 请求 | ≤ 10s |

---

### 内存占用

尽量控制：

```text
< 100MB
```

避免卡顿。

---

## 6.2 网络要求

支持：

* 蓝牙联网（通过手机）
* Wi-Fi（若设备支持）

网络异常需优雅降级。

---

## 6.3 安全要求

### API Key 安全

禁止：

```text
硬编码 API Key
```

建议：

* 使用服务端代理
* 或简单加密存储

---

### HTTPS

所有请求必须 HTTPS。

---

## 6.4 稳定性要求

要求：

* API 失败不崩溃
* 页面异常可恢复
* 数据损坏可自动重建

---

## 6.5 电量优化

由于手表设备电池有限：

避免：

* 高频轮询
* 后台长连接
* 高频动画

# 七、风险与限制

---

## 7.1 平台限制

小米 Watch S3 基于 Xiaomi Vela / HyperOS Lite，并非 WearOS，第三方能力有限。

可能限制：

* 输入法能力
* 后台网络
* 长连接
* WebSocket
* 第三方组件

---

## 7.2 网络延迟

由于通过手机蓝牙联网：

AI 回复可能较慢。

需：

* Loading 状态
* 超时处理

> 在 Xiaomi Watch S3 上实现一个轻量、稳定、低功耗的 AI Chat 应用。

* 聊天
